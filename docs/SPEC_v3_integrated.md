## 英単語学習アプリ 仕様書（統合版 v3.1）

（この文書だけで同等の再実装ができる粒度で記載）

### 1. 概要
#### 1.1 目的
参考書（単語帳）ベースで英単語を学習できるアプリ。学習効率・運用管理・監査（証跡）を両立する。

#### 1.2 対象
Webアプリ（PC/タブレット/スマホ）。実装はReact + Dexie（IndexedDB）でオフライン中心。将来クラウド同期前提。

### 2. 用語
- **参考書（Book）**：教材。章（Chapter）を持つ。UI表示は「参考書名」のみ（bookIdは表示しない）。
- **共有単語（Shared Word）**：共有ID（整数）を持つ単語。
- **自作単語（Custom Word）**：ユーザー作成。共有IDと衝突しないID体系（c:xxx等）。※自作単語の完全統合は拡張項目。
- **単語エントリ（WordEntry）**：wordId × bookId の出題/表示単位（同一単語でも参考書により訳・注釈が異なるため）。
- **ライブラリ（Library）**：ユーザー個人の保持単語集合（主に共有単語）。
- **予定（Schedule）**：1週間単位の予定。日種別（勉強/復習/テスト/総復習/休息）と範囲等を持つ。
- **予定提案（Pending Schedule）**：講師/教務/管理者が作成し、生徒が反映/拒否する案。
- **学習セッション**：学習開始〜中断/完了までのまとまり。
- **勉強時間**：学習操作イベントから算出（無操作2分以上は除外）。

### 3. ロールと所属校舎（school）
#### 3.1 ロール
- user：生徒
- teacher：講師
- manager：教務
- admin：管理者
表示名は日本語（生徒/講師/教務/管理者）。

#### 3.2 所属校舎
- school は **全員 string[]** で保持する。
- データ閲覧・通知・管理範囲はschoolで制御（UI隠しではなくロジック制御）。

#### 3.3 初期アカウント（必須）
- seito / seito / user / ["岡崎校"]
- sensei / sensei / teacher / ["岡崎校"]
- kyomu / kyomu / manager / ["岡崎校","名古屋有松校"]
- kanri / kanri / admin / ["ADMIN"]

#### 3.4 アカウント停止
- usersに isActive（1=有効 / 0=停止）を持つ。
- 停止されたアカウントはログイン不可。ログイン時に「このアカウントは停止されています。校舎長に確認して下さい。」を表示する。
- 停止は履歴を削除しない（監査ログに残す）。

### 4. UI基本方針
#### 4.1 内部ID表示禁止
- 画面上に bookId / chapterId を一切表示しない。
- 参考書は常に「参考書名（title）」、章は「第◯章」、予定は「対象名」で表示する。

#### 4.2 誤操作防止
- 危険操作（停止/拒否など）は必ず確認フローを挟む。
- 拒否時はコメント必須。

### 6. ホーム
#### 6.1 レイアウト（最新版）
縦並びで以下の順に表示する：
- **本日の予定**（ある場合のみ）
- **予定進捗**（本日の予定がある場合のみ）
- **勉強時間/目標**（常に表示。ひとかたまり）
- **今週の予定（7日）**（常に表示）
- **クイック開始**（常に表示。予定がある場合は下に来る）

#### 6.2 勉強時間/目標表示
形式：
- 「本日の勉強時間 / 目標」
- 「X分 / Y分 Z%」の1まとまり
メッセージ：
- 50%未満：Let's try!（赤）
- 50〜90%：You can do it!（赤）
- 90%以上：Good!（緑）
- 130%以上：Great!!（緑）
- 200%以上：Excellent!!!（緑）

#### 6.2.1 勉強時間の算出（v3.1確定）
- activityEvents（学習操作イベント）を時系列に並べ、
- 「隣接イベント間の時間差」を加算する
- 無操作2分以上は除外するため、各差分は最大120秒でキャップする
- 日付（dateKey）単位で集計する

#### 6.3 予定進捗
本日の予定がある場合：
- 「テスト数 / 予定の単語数 ％」
- 上記と同じ閾値でメッセージ表示
テスト数：当日の answerLogs 件数
予定の単語数：本日の schedule の rangeStart〜rangeEnd を合算（repeatCount反映）

#### 6.4 今週の予定の操作
- クリックしても即問題設定へ遷移しない。
- クリックすると **予定詳細モーダル** を表示し、内容を説明する。
- モーダル内に「この範囲を学習する」ボタンを用意し、押すと問題設定へ遷移。

### 7. 学習タブ
- 学習未開始（学習データがない）場合：
- 「『ホーム』タブや『参考書』タブから学習を開始してください。」を表示
- 「ホームへ」「参考書へ」ボタンを表示

### 8. 参考書タブ（参考書詳細）
#### 8.1 章選択
- 参考書選択後、章のチェックは基本すべてON。
- 章選択エリアの最上部に「すべてON」「すべてOFF」ボタンを置く。

#### 8.2 単語リスト兼検索
- 単語リスト兼検索窓は「章選択」の下に表示する。
- 検索対象：英語/日本語/注釈/例文

### 9. 参考書申請（生徒・講師）
#### 9.1 申請フロー（3段階）
- 参考書選択（検索可）
- 例画像表示＋写真アップロード（必須）
- 例画像と提出画像の比較＋OK/×

#### 9.2 生徒側「申請・許可待ち」リスト
- 表示対象：pending / rejected のみ
- approved は表示しない
- rejected は詳細を開ける（コメント表示）
- rejected 詳細内に「非表示」ボタン。押すとリストから消える（isHidden=1）

### 10. 申請通知（校舎・ロール）
#### 10.1 通知受信者
- 生徒(user)が申請：同校舎の講師(teacher)＋教務(manager)に通知
- 講師(teacher)が申請：同校舎の教務(manager)に通知
- 管理者(admin)：通知を受けない

#### 10.2 通知設定（講師・教務）
- 全体設定：通知をすべてOFF（ON/OFF）
- 個別設定：対象ユーザーごとに ON / OFF / 未設定
- 未設定は全体設定に従う（ただし表示上はON/OFFと同じ見た目）
- 個別設定は全体設定を上書きする
- 全体設定の下に「個別設定数」を表示
- 「すべての個別設定を解除」ボタンで全個別設定を未設定に戻す

#### 10.3 個別設定UI（ベル）
- ON：🔔（黄色）
- OFF：🔕（白黒：黒＋薄いopacity）
- 未設定：全体設定がONならON表示、OFFならOFF表示（見た目は2状態）
